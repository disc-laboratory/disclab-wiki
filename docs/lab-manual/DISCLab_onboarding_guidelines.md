# DISCLab --- Onboarding Guide

## Welcome

Welcome to the Digital Intelligence for Structures and Construction (DISC) Laboratory.

We are excited to have you join the group. This document will guide you through your first days and help you align with how we work, collaborate, and conduct research.

This is not meant to be exhaustive — you will learn many things progressively — but it provides the key elements to start effectively.

## First Day Overview

During the first day, the following topics are typically covered:

### Introduction

- Overview of the lab research vision and ongoing projects;
- Discussion of the researcher’s role and expected contributions;
- Introduction to collaborators and current work;
- Setup of required accounts and platforms.

### Accounts and platforms

Please ensure you have access to:

- GitHub accounts
- OrcId account
- ResearchGate account
- Google scholar account
- LinkedIn account (recommended)

## Key Platforms

### Lab Website

The [lab website](https://disclab.pantoja-rosero.com/) contains all relevan information related to the DISCLab research group. This includes the groups research vision, research areas, team members, publications, news and positions available.

Each member will be added to the lab website with:

- a profile photo;
- contact info, a short research description or interests.

### GitHub (DISC Lab)

All code development is managed through the lab [GitHub organization](https://github.com/disc-laboratory).

- Version control must be used consistently;
- Code should be pushed regularly;
- Repositories should be clean and documented.

### DISCLab wiki

The [Lab's Wiki page](https://disc-laboratory.github.io/disclab-wiki/) contain relevant information useful for the team members. It is encoraged to review the content as well to contribute to it. The objective of this page is to have a central knowledge base of the group for any possible operational aspect. This involves guideliness, templates, tutorials, best practices, etc.

### Youtube

[YouTube channel](https://www.youtube.com/@disc-lab) is used to share videos related to research methodologies developed in the group.

- It is encouraged to prepare a short video presenting the methodologies and showcasing case of studies;
- Do this for journal or conference papers.

### LinkedIn

[LinkedIn page](https://www.linkedin.com/company/disc-lab/) is used to communicate the group's research activities.

- Members are encouraged to maintain a profile;
- Participation is optional but recommended.

### ResearchGate

[ResearcgGate page](https://www.researchgate.net/lab/Digital-Intelligence-for-Structures-and-Construction-DISC-Lab-Bryan-German-Pantoja-Rosero) is used to dissiminate the group's research publications.

- Once a paper is published, a researchgate post is expected;
- Provide the full paper (if not possible due to copyright, publish the latest submited manuscript) together with figures.

## Profile Information

New members are requested to provide:

- a natural professional photo (no heavy editing);
- a short research bio (3--5 lines);
- current affiliation and role.

This will be used for:

- Lab website
- Internal records
- Research visibility

## Working Structure

All research activities are organized using a project-based structure.

- The main workspace is located in `02_Projects/`;
- Each project follows a standardized internal structure;

Each project contains its own README files explaining how to organize work.

## Code and Research Practices

### Code

- Core code must be stored in GitHub;
- Important code should not remain only locally;
- Code should be clear and maintainable.

### Experiments

- Experiments must be documented;
- Keep track of: configurations, datasets, and results (should be traceable).

### Data

- Raw data must not be modified;
- Data structure must be followed consistently.

## Research Materials

You will receive:

- Relevant project documents (proposals, notes, datasets)
- Access to existing code repositories
- References to relevant literature

## Communication

### Meetings

- Regular meetings will be scheduled (weakly: individual and group);
- Members are expected to communicate progress and challenges clearly;

### Expectations

- Communicate clearly and early;
- Ask questions always (even if it is the simplest);
- Be proactive in your work.

## Available Resources

You will have access to:

- Lab templates (reports, presentations, code)
- Internal documentation (Lab Manual)
- Project-specific materials

## First Week Goals

By the end of the first week, members should:

- understand their project scope;
- have access to all required tools;
- be familiar with the folder structure;
- be able to run or explore existing code;
- begin initial tasks.

## Final Remarks

- Do not worry about understanding everything immediately
- Focus on building a clear understanding step by step
- Good organization and documentation are essential in this lab
