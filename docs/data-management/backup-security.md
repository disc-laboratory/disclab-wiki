# Backup and Security

## Backup principles

- use institutionally approved storage for canonical copies
- maintain at least one additional backup for critical assets
- do not keep the only copy on a laptop or external drive

## Security principles

- restrict access by project need
- remove access when members leave the project
- never publish sensitive data without approval
- document external sharing of large datasets

## Suggested practice

- primary storage on managed institutional infrastructure
- mirrored backup for critical project data
- encrypted storage for sensitive material when applicable
